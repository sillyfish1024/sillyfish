IDEA中Maven依赖无法正常下载

发表于2021-04-17|更新于2021-04-17|tools
|阅读量:

问题描述

在IDEA使用maven下载依赖时出现错误，控制台错误如下：

Could not transfer artifact org.springframework:spring-context-support:pom:5.2.8.RELEASE from/to alimaven (http://maven.aliyun.com/nexus/content/groups/public/): Transfer failed for http://maven.aliyun.com/nexus/content/groups/public/org/springframework/spring-context-support/5.2.8.RELEASE/spring-context-support-5.2.8.RELEASE.pom

分析

在本地仓库发现只有名为：spring-context-support-5.2.8.RELEASE.pom.lastUpdated的文件，通过URL发现可以直接访问下载，说明仓库和仓库配置没有问题；最终确认是由于JVM校验SSL证书时出现错误，无法找到证书，可以在IDEA的日志中找到以下描述：

….
Caused by: java.lang.RuntimeException: sun.security.validator.ValidatorException: PKIX path building failed: sun.security.provider.certpath.SunCertPathBuilderException: unable to find valid certification path to requested target
at java.base/sun.security.validator.PKIXValidator.doBuild(PKIXValidator.java:439)
at java.base/sun.security.validator.PKIXValidator.engineValidate(PKIXValidator.java:306)
…

解决

找到了问题的原因，解决方向不外乎两个：

  • 在JVM添加中央仓库的SSL证书
  • 忽略SSL证书校验

首先尝试了添加的证书的方式（过程参考：java - “PKIX path building failed” and “unable to find valid certification path to requested target” - Stack Overflow），但并没有效果，也没有发现原因所在，便采用忽略SSL证书校验的方式，问题最终解决，依赖成功下载，在IDEA（版本:2020.3.3）可以如下设置：

参数：-Dmaven.wagon.http.ssl.insecure=true -Dmaven.wagon.http.ssl.allowall=true -Dmaven.wagon.http.ssl.ignore.validity.dates=true
